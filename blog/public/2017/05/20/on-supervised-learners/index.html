On Supervised Learners

2017-05-20 · 359 words · 2 minute read
Supervised learning · ML foundations

The mission statement of this blog is to discuss the foundations of machine learning using elegant mathematics. In order to do that, the natural starting point seems to be to tackle the problem of devising an all-encompassing mathematical definition of machine learning which is robust enough to allow us to prove theorems and by extension explain the some of the current mysteries of the field. Unfortunately, machine learning being as vast field as it is, this seems to be a little quixotic of a pursuit for the time being…

Instead, we choose to be a little more pragrmatic and focus on a slightly more specific subfield: that of supervised learning.
Recall that the paradigm in supervised learning is to start with a dataset split into features and labels and ask wether we can find an assignment between features and labels (of a certain type depending on the context) that best matches the given dataset.

Let’s dissect that sentence a little more…

We’ll denote the set of features and labels by $\mathfrak{X}$ and $\mathfrak{y}$ respectively. The space of all possible datasets $\mathfrak{D}$ should in turn consist of a collection of sets (we’ll denote them by $\Delta$) each of which is a subset of $\mathfrak{X}\times \mathfrak{y}$ (also known as a relation between $\mathfrak{X}$ and $\mathfrak{y}$).
We next introduce a space $\mathfrak{H}\subset \mathfrak{y}^\mathfrak{X}$ of hypotheses which contains the functions $\mathfrak{X}\longrightarrow \mathfrak{y}$ that describe the possible ways we wish to assign a label to a feature.
A supervised learner now constructs an assignment from the dataspace to the hypothesis space $$h:\mathfrak{D}\longrightarrow \mathfrak{H}: \Delta\mapsto h_\Delta$$ such that $h_\Delta$ “describes” the data $\Delta$ optimally.
To formalize this idea, we introduce a cost function which assigns a real number given any choice of data and hypothesis: $$c: \mathfrak{D}\times \mathfrak{H}\longrightarrow \mathbb{R}: (\Delta,f)\mapsto c(\Delta,f)$$ The idea that the choice of hypothesis $h_\Delta$ fits the data best now translates to $h_\Delta$ minimizing the cost: $$ c(\Delta,h_\Delta)=\min_{f\in \mathfrak{H}} c(\Delta,f) $$ This leads to the following definition:

A learning algorithm consists of a tuple $(\mathfrak{X},\mathfrak{y},\mathfrak{D},\mathfrak{H},h,c)$ where $\mathfrak{D}\subset \textrm{Rel}(\mathfrak{X},\mathfrak{y})$, $\mathfrak{H}\subset \mathfrak{y}^{\mathfrak{X}}$ and $h:\mathfrak{H}\longrightarrow \mathfrak{D}$, $c: \mathfrak{D}\times \mathfrak{H}\longrightarrow \mathbb{R}$ are functions satisfying the learning condition $$ c(\Delta,h_\Delta)=\min_{f \in \mathfrak{H}} c(\Delta,f) $$ is satisfied
